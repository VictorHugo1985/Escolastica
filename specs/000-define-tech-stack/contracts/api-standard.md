# Contract: API Communication Standards

## REST API Pattern

All endpoints exposed by the Nest.js backend must follow these standards:

- **Path Naming**: kebab-case (e.g., `/academic-sessions`).
- **Methods**:
  - `GET`: Retrieve data (no side effects).
  - `POST`: Create new resources.
  - `PUT`: Update existing resources (full replacement).
  - `PATCH`: Partial update of a resource.
  - `DELETE`: Remove a resource.

## Error Handling

Standardized JSON error responses:

```json
{
  "statusCode": 400,
  "message": "Validation failed",
  "error": "Bad Request",
  "details": [
    {
      "field": "email",
      "issue": "Invalid format"
    }
  ]
}
```

## Security

- **Authentication**: Bearer Token (JWT generated by Nest.js `AuthModule`) in the `Authorization` header. Authentication is managed locally — Supabase Auth is NOT used.
- **Validation**: All incoming data must be validated against a Zod schema before reaching the service layer.

## Documentation

- Every controller and DTO must include Swagger decorators (`@ApiTags`, `@ApiOperation`, `@ApiResponse`).
- Auto-generated Swagger UI available at `/api/docs`.
